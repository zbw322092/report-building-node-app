# Be BASIC
Just thinking about, What is the core of scroll components, without reading any code?
Yes, it is SCROLL.
HOWEVER, You Guess What.
Browsers can scroll automatically, we need do nothing. It is a truth everybody knows.
So, now the question is, Why we need scroll components?
Yes, sometimes, we need scroll to a specified postion! ( `y` 400px maybe )
But, native method window.scrollTo() can deal with that.
Well, but we need SMOOTH scroll! window.scrollTo() is not enough!
No, we have smooth-behavior css property can make scroll smoothly. 
      
        scroll-behavior: smooth;
Ok, but, what about scrolling to a specified HTML element?
Also, native method can help us. It is Element.scrollIntoView() 
      
          var element = document.getElementById("box");
          
          element.scrollIntoView();
          element.scrollIntoView(false);
          element.scrollIntoView({block: "end"});
          element.scrollIntoView({behavior: "smooth", block: "end", inline: "nearest"});
You can even config smooth scroll in Element.scrollIntoView() method.
Now we know native methods will help us, are these method perfect?
